# 기술 선택 근거 문서

## GUI 프레임워크: PyQt5

- **선택 이유**: PyInstaller와의 호환성이 안정적이며 국내 레퍼런스가 풍부함
- **대안**: PyQt6 (더 현대적이나 패키징 이슈 가능성 있음)
- **결정일**: Phase 1

---

## AI 프레임워크: PyTorch + Ultralytics YOLOv8

- **선택 이유**: 오픈소스, 실시간 객체 탐지에 최적화, 전이학습 지원, 활발한 커뮤니티
- **대안**: TensorFlow, OpenCV DNN
- **결정일**: Phase 1

---

## 카메라 캡처: OpenCV (cv2) + threading

- **선택 이유**: Python 환경에서 가장 널리 사용되는 영상처리 라이브러리. threading으로 멀티카메라 동시 처리 구현
- **대안**: PyQt5 QThread 단독 사용 (추후 리팩토링 고려)
- **결정일**: Phase 2

---

## 카메라 영상 포맷: MJPG

- **선택 이유**: USB 2.0 프로토콜 기반 C920 카메라 4대 동시 운용 시 대역폭 초과 문제 해결
  - YUY2(비압축) 대비 전송 데이터량 약 1/10 수준으로 감소
  - 640x480 @ 30fps 4대 동시 운용 가능
- **대안**: YUY2 (비압축, 화질 우수하나 대역폭 초과)
- **결정일**: Phase 2

---

## 카메라 초기화 방식: 병렬 threading

- **선택 이유**: 순차 초기화 시 카메라 1대당 약 2~3초 소요 → 4대면 8~12초
  - 병렬 초기화로 전체 시간을 카메라 1대 수준으로 단축
- **결정일**: Phase 2

---

## 데이터베이스: SQLite (예정)

- **선택 이유**: 별도 서버 불필요, 단독 실행 파일에 포함 가능
- **대안**: MySQL, PostgreSQL
- **결정일**: Phase 6에서 확정 예정

---

## 패키징: PyInstaller

- **선택 이유**: Python 앱을 Windows .exe로 변환, 무료 오픈소스, PyQt5와 안정적 호환
- **대안**: cx_Freeze, Nuitka
- **결정일**: Phase 1
